# 贡献指南 · Contributing Guide

感谢你为 Campus 2027 贡献信息！本项目通过 **GitHub Issue 表单** 收集数据，
维护者审核后由 GitHub Actions 自动更新 README。

---

## 如何贡献

### 1. 新增一条招聘信息

点击 [Issues → New Issue](../../issues/new/choose)，选择 **「新增招聘信息」**。

填写以下字段：

| 字段 | 说明 | 是否必填 |
| ---- | ---- | -------- |
| 公司名称 | 公司全名，如：字节跳动 | ✅ |
| 投递链接 | 官方校招投递页面 URL | ✅ |
| 批次类型 | 提前批 / 正式批 / 实习 | ✅ |
| 行业分类 | 从下拉菜单选择 | ✅ |
| 招聘状态 | 已开放 / 未开放 / 已截止 | ✅ |
| 开放日期 | 格式 YYYY-MM-DD | 可选 |
| 预计开放日期 | 格式 YYYY-MM-DD | 可选 |
| 工作地点 | 用逗号分隔 | 可选 |
| 备注 | 内推码、截止时间等 | 可选 |

### 2. 更新已有信息

选择 **「更新招聘信息」** 模板，填写**公司名称 + 批次类型**（必须与现有记录完全一致）及需要更新的字段。

### 3. 标记已截止

选择 **「标记已截止」** 模板，填写**公司名称 + 批次类型**即可。

## 审核流程

```
提交 Issue  →  维护者审核  →  打 approved 标签
     ↓                              ↓
  等待中              GitHub Actions 自动更新 listings.json + README
                                    ↓
                           自动关闭 Issue，回复确认
```

- 维护者可能会打 `needs-info` 要求补充信息
- 重复条目会被打上 `duplicate` 并关闭

---

## 数据规范

- **信息来源**：只接受来自公司官网、官方公众号的一手信息
- **链接**：必须是直接可访问的投递页面，不要用缩短链接
- **公司名称**：与已有记录保持一致（避免「字节」vs「字节跳动」的歧义）
- **内推码**：欢迎附上，但不强制

## 本地开发

```bash
# 克隆仓库
git clone https://github.com/<你的用户名>/Campus2027.git
cd Campus2027

# 安装依赖（无外部依赖，Python 3.9+ 即可）
python --version  # >= 3.9

# 手动刷新 README
cd .github/scripts
python update_readme.py

# 验证输出
cat ../../README.md
```
